## What Ralph does

Ralph runs Claude Code in iterative loops with fresh context per iteration. It shells out to the `claude` CLI for all LLM work — there is no SDK dependency.

## Key architecture

- **Session files** — all state lives in markdown files under `$TMPDIR/ralph/session-<id>.md`. Front matter tracks stage and iteration counts.
- **Claude CLI shelling** — `src/claude.ts` wraps `execa('claude', ...)` for both interactive and non-interactive modes. Non-interactive uses `--print`.
- **Flow** — `src/flow.ts` orchestrates the full pipeline: init → refine → plan → execute loop. `runNonInteractive` is the headless entry point used by `ralph run`.
- **Session slugs** — session IDs are human-readable slugs generated by Claude Haiku from the task description (e.g. `weather-dashboard-react`). Falls back to hex if generation fails.

## Testing inside Claude Code

Ralph shells out to `claude`, so running it inside a Claude Code session will fail. See `CONTRIBUTORS.md` for the `unset CLAUDECODE` workaround.
